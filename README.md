# Design patterns
**Design patterns** are typical solutions to commonly occuring software design problems. 

Design patterns usually have the following sections in their describtion:
- **Intent** of the pattern
- **Motivation** 
- **Structure** 
- **Code example**

Main design patterns groups:
- **Creational patterns** - provide objec creation mechanisms to increase flexibility and reuse of existing code.
- **Structural patterns** - explain how to assemble objects and classes into larger structures, while preseving flexibility and efficiency.
- **Behavorial patterns** - take care of effective communication and responsibilty assignments between objects.
